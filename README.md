## Project Overview

The game presents a hand of Mahjong tiles. The player bets whether the **next** hand will have a **higher** or **lower** total value. After each round the honor tiles (winds & dragons) shift their values based on the outcome, influencing future rounds. A leaderboard is persisted in the browser‚Äôs `localStorage`.

## Game Rules & Edge‚ÄëCase Logic

1. **Betting** ‚Äì The player chooses **higher** or **lower**. The next hand is drawn and its total compared to the current hand.
2. **Honor tile value shift** ‚Äì After a win, each honor tile‚Äôs value is increased by‚ÄØ1; after a loss it is decreased by‚ÄØ1. Number tiles keep their face value.
3. **Score calculation** ‚Äì When the player wins, the round score equals the **sum of all honor tile values** (not the hand total). This is calculated by `calculateHonorValuesTotal`.
4. **Tie handling** ‚Äì If the current hand and the next hand have the **same total points**, the round is **skipped**. No points are awarded and honor values do not change. The round is **not** considered a loss.
5. **Automatic reshuffle & round skip** ‚Äì If the draw pile contains **fewer tiles than the configured hand size**, the deck is automatically reshuffled (combining the discard pile and a fresh deck) and the round is **skipped**. This logic lives in `autoSkipShortRounds`.
6. **Game end conditions** ‚Äì The game ends when a tile value reaches **0 or 10** (tile limit) or after **three deck exhaustions** (deck limit).

## Implementation Details

* **Deck creation** ‚Äì `createFreshDeck` builds a full set of tiles based on configurable copies per category (numbers, winds, dragons). Tiles are shuffled with `shuffleTiles`.
* **Honor values** ‚Äì Stored in a map (`honorValues`) keyed by `"kind:label"`. `applyHonorValuesToHand` injects the current honor values into a hand for rendering.
* **Round resolution** ‚Äì `resolveRound` draws the next hand, evaluates the bet, updates honor values, calculates points, and determines whether a reshuffle or game‚Äëover condition occurred.
* **Automatic short‚Äëround skip** ‚Äì Implemented in `autoSkipShortRounds`. When the draw pile is too small, the deck is reshuffled, the round counter is incremented, and a toast informs the player.
* **Leaderboard persistence** ‚Äì Stored as JSON under the key `hand-betting-game.leaderboard` in `localStorage`. The top‚ÄØ5 scores are kept.
* **Settings persistence** ‚Äì Hand size and copies per category are saved under `hand-betting-game.settings`.
* **UI components** ‚Äì `HonorPill`, `TileCard`, `SectionHeading`, etc., are thin wrappers that apply Tailwind classes and optional Radix UI primitives.

## Assumptions & Limitations

* The game assumes a **fixed hand size** (default‚ÄØ4) but can be changed via the settings panel.
* Honor tile values are bounded between **0 and 10**; reaching either bound ends the game.
* Currently a **tie** (equal hand totals) is treated as a loss; the documented rule to skip the round on a tie is a planned improvement.
* All persistence is client‚Äëside; no backend or multiplayer support.
* The UI is designed for desktop browsers; touch‚Äëspecific optimisations are minimal.
